1.3 INTRODUCCIÓN

En el desarrollo web moderno, crear experiencias interactivas mejora la motivación y el aprendizaje. Duelo de Preguntas es un juego web de preguntas y respuestas por turnos, donde los jugadores compiten respondiendo correctamente para acumular puntos.

En esta guía se presenta el funcionamiento del juego, la estructura del proyecto y la conexión con una base de datos en la nube para registrar puntajes y mostrar resultados. Se aplican conceptos de HTML, CSS y JavaScript para construir una interfaz dinámica y fácil de usar.

1.4 OBJETIVOS DE LA GUÍA

Al finalizar la práctica, el estudiante será capaz de:

  • Diseñar una interfaz clara y responsiva para un juego web.
  • Implementar la lógica del duelo (turnos, puntajes y validación de respuestas).
  • Mostrar preguntas con opciones múltiples y retroalimentación inmediata.
  • Registrar resultados y mostrar historial o ranking usando Supabase.

  • Valida respuestas, calcula puntajes y determina un ganador.

1.7 FUNDAMENTO TEÓRICO

Duelo de Preguntas (Quiz Battle)

Duelo de Preguntas es un juego interactivo de preguntas por turnos. Cada ronda muestra una pregunta con opciones múltiples; el jugador elige una respuesta y el sistema valida si es correcta, actualiza el puntaje y cambia el turno. Al final, se muestra el ganador y el resumen de resultados.

Características principales:
  • Turnos alternados entre jugadores.
  • Puntaje acumulativo y definición de ganador.
  • Interfaz clara con retroalimentación visual.
  • Registro de resultados para historial o ranking.

Lógica del juego y reglas

Turnos

El juego asigna un turno por jugador. Al responder, el sistema valida la opción y luego cambia el turno automáticamente para mantener la dinámica justa.

Puntaje

Cada respuesta correcta suma puntos. Las incorrectas no suman (o pueden restar, si así se configuró). El objetivo es obtener el mayor puntaje al finalizar.

Preguntas y categorías

Las preguntas pueden agruparse por categorías o dificultad. Esto permite una experiencia más organizada y adaptable a distintos temas.

Resultados y ranking

Al terminar la partida, se muestran puntajes y ganador. Los resultados pueden guardarse para consultar historial y crear un ranking de los mejores jugadores.

Experiencia de usuario: 3 pilares

Claridad

Interfaz simple: pregunta visible, opciones claras y botones fáciles de usar.

Velocidad

Respuestas rápidas: validación inmediata y transición fluida entre turnos.

Diversión

Competencia por puntos, retroalimentación visual y sensación de duelo entre jugadores.
